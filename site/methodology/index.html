VALIDATION

Out-of-sample accuracy. Updated daily.

Every regime_calls row is written on the pipeline date (no backdating). The next trading day, validation_regime.py compares the prior call’s directional read to realised 1D FX returns and upserts validation_log. When the predicted direction is NEUTRAL, correct_1d is intentionally left null — those days do not count toward hit-rate numerators or denominators, which keeps early-sample dashboards honest when the composite sits flat. Rolling accuracy on the public shell is shown only after enough evaluated (non-null) rows exist; until then you will see an explicit “validation accumulating” notice rather than a fake percentage.
